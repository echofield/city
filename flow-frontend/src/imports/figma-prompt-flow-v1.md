# Final Figma Prompt for FLOW V1

## Context

FLOW is a **mobility intelligence interface for professional drivers**.

It helps drivers **anticipate demand waves in the city**.

The design must feel:

• minimal
• calm
• strategic
• readable in <2 seconds while driving

Reference inspirations:

* **Waze simplicity**
* **Bloomberg Terminal clarity**
* **Arché radar exploration**
* **dark minimal OS interfaces**

Primary color palette:

```
Black background
Emerald green signals
Soft grey structure
Occasional gold/yellow for alerts
```

---

# Navigation Structure (Bottom Mobile)

Order:

```
CARTE
LIVE
RADAR
SEMAINE
```

Why:

```
CARTE = spatial overview
LIVE = immediate action
RADAR = exploration
SEMAINE = strategic planning
```

---

# 1. LIVE (action screen)

Purpose:

**what should I do right now**

Layout:

Large directive headline.

Example:

```
REJOINS OPERA
COTE BOULEVARD DES CAPUCINES
```

Under:

```
L'Olympia
Sortie concert
22:20 – 23:00
```

Then small info:

```
distance
estimated € range
window closing countdown
```

Below:

```
CONFIRMER
NAVIGUER
```

Under that:

**2–3 alternative signals only**

Not more.

Drivers must **not scroll endlessly**.

---

# 2. CARTE (territory awareness)

Purpose:

**where pressure is forming**

Map of Paris arrondissements.

Zones glow when pressure rises.

Example:

```
XI
IX
XVIII
```

Glow intensity shows demand.

Right panel shows **selected signal**.

Information blocks:

```
OPPORTUNITY SCORE
COMPETITION
ESTIMATED €
```

One big action:

```
NAVIGUER
```

Keep map **clean and minimal**.

No clutter.

---

# 3. RADAR (city scan)

Radar must **NOT look like Live**.

Radar is **exploration mode**.

Inspired by **Arché radar**.

Layout:

List of nearby signals by **timeline**.

Example:

```
NOW
Theatre du Chatelet
~1000 people leaving
PIC 22:10

+10 min
Bataclan
concert ending

+15 min
Palais Garnier
opera exit
```

Each row shows:

```
place
crowd estimate
time of peak
signal score
```

Driver taps a row → opens signal.

Radar answers:

```
what is about to happen nearby
```

This makes drivers **proactive**.

---

SEMAINE FORTE

Drivers use this **before their shift**.

# Key Design Rule

Everything must answer **one question only**:

```
Where should I be?
```

Not:

```
look at data
```

Drivers don’t want dashboards.

They want **direction**.
