# 1. CARTE should be first

The map is the **natural entry point** for drivers.

Drivers think spatially:

```
Where am I
Where is demand
Where should I move
```

When opening Flow, they instinctively want to see:

```
city + signals
```

Your map already does that very well.

So **CARTE = situational awareness**.

---

# 2. LIVE second (action)

After seeing the map, the driver wants:

```
What should I do right now?
```

That’s exactly what LIVE does.

LIVE is basically:

```
recommended move
```

Example:

```
REJOINS OPERA
COTE BOULEVARD DES CAPUCINES
```

So LIVE belongs **right after the map**.

---

# 3. RADAR third (exploration)

Radar is not urgent.

It is more like:

```
Let me see what's forming around
```

Drivers will open it when:

• waiting
• exploring opportunities
• curious about signals

So Radar belongs **after action screens**.

---

# 4. SEMAINE last (planning)

Weekly planning is the least frequently used.

Drivers might check it:

• before starting a shift
• in the afternoon
• to plan weekend strategy

So it naturally belongs **last**.

---

# 5. Final navigation structure

Your proposed structure becomes:

```
CARTE   → spatial awareness
LIVE    → immediate action
RADAR   → discovery
SEMAINE → future planning
```

This matches the **mental workflow of a driver**.

The old order:

```
LIVE | RADAR | CARTE | SEMAINE
```

started with **instructions before context**.

But drivers prefer:

```
context → decision
```

Not:

```
decision → context
```
