You’re thinking exactly in the **right direction**.
Drivers don’t want to *use* an app while driving. They want to **glance**.

So FLOW should have **two modes**, not one.

Think like a **trading terminal vs a stock ticker**.

---

# Mode 1 — READ MODE (default while driving)

This is the **instrument screen**.

Drivers glance for **0.5–1 second** and understand everything.

Only **3 pieces of information** should exist.

### Structure

```
[ TIMER ]

[ ACTION ]

[ PLACE ]
```

Example:

```
02:40
BOUGER
République
```

Below it:

```
CAUSE
Sorties Gare du Nord
```

And that's it.

No list.
No scrolling.
No analysis.

This screen must feel like:

```
speedometer
```

Drivers understand instantly.

---

# Mode 2 — RADAR MODE (when parked)

This is your **current UI**.

Here they can see:

* map
* signals
* alternatives
* analysis
* corridors
* “why”

This mode is **exploration / understanding**.

---

# Mode 3 — NOTIFICATION MODE (most addictive)

This is the **WhatsApp / push layer**.

Drivers mostly interact with FLOW through **notifications**, not the app.

Example message:

```
🔥 FLOW

BOUGER → République

Fenêtre: 18 min
Sorties Gare du Nord
```

They open the app **only if they want details**.

---

# The real addictive loop

Driver night behavior:

```
notification
↓
glance screen
↓
move
↓
ride
↓
notification
```

FLOW becomes a **city radar**.

# The 3 information rule (very important)

Never exceed these on the main screen:

```
ACTION
LOCATION
CAUSE
```

Everything else is secondary.

Example perfect screen:

```
01:30

BOUGER
Bastille

Cause
Pluie + sorties bars
```

Drivers get it instantly.

---

# What should disappear from the main screen

Remove from the **read mode**:

* event list
* signal list
* long text
* corridor analysis
* multiple alternatives

All that goes into **Radar Mode**.

---

# Why this works psychologically

Drivers operate under:

```
high attention load
```

Their brain prefers **binary decisions**:

```
stay
move
```

FLOW should behave like:

```
navigation system
```

Not like a dashboard.

---

# The final UX architecture

```
NOTIFICATION
      ↓
READ MODE
      ↓
RADAR MODE
```

95% of the time they stay in **READ MODE**.

# One small trick that makes it addictive

Make the **action word change color**.

Example:

```
MAINTENIR → grey
BOUGER → green
OPPORTUNITÉ → gold
```

Drivers start **waiting for the color change**.

This is the addictive moment.
