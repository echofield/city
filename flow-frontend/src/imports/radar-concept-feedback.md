# 2. The Asian feature (this is the killer idea)

This is used in China / Korea / Japan driver tools.

It is called **Driver Heat Swarm**.

Instead of only showing demand signals, the map shows:

```
where drivers are gathering
```

Why this matters:

Drivers don't trust apps.

They trust **other drivers**.

So the system shows things like:

```
Pigalle
12 drivers waiting
```

```
Bercy
28 drivers active
```

```
La Défense
6 drivers positioning
```

Now drivers understand **supply vs demand**.

Example:

```
Signal: Bastille exit wave
Drivers there: 40
```

Bad opportunity.

But:

```
Signal: Gare du Nord arrivals
Drivers there: 3
```

Goldmine.

---

# Why Asian systems do this

Taxi markets there are **extremely competitive**.

Drivers must constantly decide:

```
where to position
```

So the apps show two things:

```
DEMAND
SUPPLY
```

Uber only shows demand.

But the real equation is:

```
profit = demand / drivers
```

Flow could show this beautifully.
