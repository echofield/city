1. LIVE vs RADAR (philosophical difference)

LIVE is:

The system tells the driver what to do.

RADAR is:

The driver explores the environment.

So the UI must communicate this difference.

Mode	Role	Feeling
LIVE	directive	confident
RADAR	exploratory	calm

Right now Radar still feels directive.

2. Radar should not give commands

In your screenshot Radar still shows things like:

REJOINS OPERA
ANTICIPE

Those belong in LIVE.

Radar should show signals, not instructions.

Example Radar entry:

Théâtre du Châtelet
sortie 22:45
≈ 800 personnes

or

Restaurant cluster
rush 23:30

Let the driver decide.

3. Radar should feel lighter visually

Right now Radar has the same strong green action language as LIVE.

But Radar should use much softer visuals.

Example hierarchy:

white / grey text
small accent colors
very minimal

Only highlight when the user clicks.

Think of Radar like:

sonar

not

alert system
4. Radar interaction (your ARCHÉ idea)

Your instinct here is excellent.

Radar should behave like tap → reveal environment.

Example interaction:

Driver taps:

Théâtre du Châtelet

Then panel opens:

Sortie prévue 22:45
≈ 900 personnes

Nearby signals:
restaurants
bars
taxi pickup spots

Exactly like ARCHÉ:

place → context → story

But here it’s mobility context.

Tap → see details.

ARCHÉ works because it creates:
