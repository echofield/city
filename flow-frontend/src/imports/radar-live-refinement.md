We keep the current LIVE layout, but we want to inject the visual philosophy of the old RADAR screen.

The radar screen has three qualities we want to bring into LIVE:

• more breathing space
• stronger focus on the main command
• calmer visual rhythm

Do not redesign the product.
This is a refinement, not a new layout.

1 — Keep the current LIVE structure

Structure stays:

City pressure bars (top)

Main signal

Secondary signals

Navigation CTA

No structural change.

2 — Increase vertical breathing space

The radar screen feels better because it has large vertical spacing.

Apply:

• +30–40% vertical padding around the main signal card
• more empty space above and below the main instruction
• reduce visual density between blocks

Goal: make the main signal feel important.

3 — Make the command typography more dominant

Take inspiration from RADAR.

The command should visually dominate.

Example:

RESTE EN POSITION
COTE PLACE DU CHATELET

Improvements:

• increase font size of command by ~20–25%
• slightly increase letter spacing
• keep strong neon green color

The command must be readable in one glance while driving.

4 — Simplify the main signal card

The radar version works because it shows only essential information.

Main card should visually prioritize:

1️⃣ Command
2️⃣ Location
3️⃣ Wave timing

Everything else becomes secondary micro info.

Reduce visual weight of:

• metadata labels
• small explanatory text

5 — Introduce radar-style calm spacing

Between elements inside the main card:

Increase spacing between:

Location
Command
Signal details
CTA

Radar screen feeling = clear blocks separated by space.

6 — Maintain the Flow design system

Do NOT change:

• color palette
• neon green glow
• typography family
• card system
• navigation tabs

Only adjust spacing, hierarchy, and focus.

7 — Goal of this change

LIVE must feel like:

A calm tactical command interface

Not like:

A dense data feed

Drivers should instantly read:

WHAT TO DO
WHERE
WHEN
