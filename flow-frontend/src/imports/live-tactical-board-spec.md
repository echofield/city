The current LIVE implementation is too generic and does not surface the intelligence of the backend clearly enough.

We need to move toward the stronger direction shown in the reference:
- specific place names
- concrete causes
- visible timing
- stronger hierarchy
- expandable tactical detail

Changes needed:

1. Replace generic labels like:
- SIGNAL
- Flux passagers
- Signal fort

with concrete signal naming:
- CDG — départs matinaux
- Gare du Nord — arrivées trains
- Olympia — sortie concert
- Bastille — restaurants fermeture
- Pigalle — sortie clubs

2. Improve card hierarchy
Collapsed card should show:
- place
- cause
- timing / countdown
- one-line action or posture
- confidence / score if useful

3. Make cards expandable
On click / tap, expand the card to reveal tactical details:
- why the signal exists
- event / venue involved
- time window
- expected ride type
- recommended positioning
- alternative nearby zone
- confidence factors if available

4. Keep LIVE visually premium and fast to scan
Do not flatten all cards into the same generic style.
Top signals should feel stronger.
Concrete information should replace abstract phrasing.

5. Use the backend richness
We already have restaurants, theatres, venues, airports, nightlife, banlieue events.
Surface that intelligence directly instead of abstracting it too early.

Goal:
LIVE should feel like a professional tactical board, not a generic signal list.
